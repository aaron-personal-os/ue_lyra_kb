# 路线图

## 目标

从干净 Unreal 工程出发，逐步搭建一个单机 3D 动作技能型 roguelike 的可玩原型。路线优先保证“技能释放闭环”和“局内成长闭环”，避免早期陷入过度架构。

## 阶段 0：项目初始化

目标：建立干净工程和基础约定。

- [ ] 创建 Unreal Third Person 模板工程。
- [ ] 启用 GameplayAbilities、GameplayTags、GameplayTasks 插件。
- [ ] 建立基础模块命名约定。
- [ ] 创建基础 C++ 角色类和 PlayerController。
- [ ] 确认是否使用 Enhanced Input。
- [ ] 建立最小日志分类和调试命令。

验收标准：

- 能在编辑器里运行第三人称角色。
- 工程能正常编译。
- 输入、移动、相机正常。

## 阶段 1：GAS 最小技能闭环

目标：玩家能释放第一个技能，并通过 GAS 处理冷却、消耗和命中效果。

- [ ] 给玩家角色添加 Ability System Component。
- [ ] 创建基础 AttributeSet：Health、Energy、AttackPower。
- [ ] 创建第一个 GameplayAbility。
- [ ] 创建冷却 GameplayEffect。
- [ ] 创建消耗 GameplayEffect。
- [ ] 创建伤害 GameplayEffect。
- [ ] 用 GameplayTag 标记冷却、状态和技能类型。
- [ ] 在 UI 或日志里显示技能释放、失败原因、冷却状态。

验收标准：

- 玩家按键可以释放技能。
- 能量不足时技能失败。
- 冷却期间技能失败。
- 技能命中敌人后造成伤害。

## 阶段 2：敌人与战斗房间

目标：建立最小战斗回路：进房间、刷敌人、清房、结算奖励。

- [ ] 创建基础敌人角色。
- [ ] 敌人拥有生命属性。
- [ ] 创建基础 AI 行为：追逐、攻击、死亡。
- [ ] 创建 RoomRuntimeController。
- [ ] 进入房间时刷怪。
- [ ] 清空敌人后标记房间完成。
- [ ] 生成一个奖励选择界面或临时奖励交互。

验收标准：

- 玩家进入房间后敌人生成。
- 消灭所有敌人后房间完成。
- 房间完成后能获得一次奖励。

## 阶段 3：Relic / 词条系统

目标：让 roguelike 的局内构筑开始成立。

- [ ] 创建 RelicData 数据资产。
- [ ] 支持 relic 授予无限期 GameplayEffect。
- [ ] 支持 relic 授予 AbilitySet。
- [ ] 创建基础奖励池。
- [ ] 清房后从奖励池随机抽取 3 个 relic。
- [ ] 玩家选择 relic 后立即生效。
- [ ] 在 RunManager 中记录已获得 relic。

验收标准：

- 拾取“伤害 +10%”类 relic 后，后续技能伤害变化。
- 拾取“获得新技能”类 relic 后，玩家可以释放新技能。
- 本局内 relic 列表能被查询和显示。

## 阶段 4：RunManager 与楼层推进

目标：建立一局 run 的状态管理。

- [ ] 创建 RunManagerSubsystem。
- [ ] 支持 StartNewRun(seed)。
- [ ] 保存当前楼层、房间索引、已访问房间。
- [ ] 保存玩家本局 relic、临时货币、钥匙等资源。
- [ ] 支持进入下一房间。
- [ ] 支持进入下一楼层。
- [ ] 支持结束 run 并生成结算数据。

验收标准：

- 一局 run 可以从第 1 层开始推进到下一层。
- relic、货币等本局状态在房间切换后保留。
- run 结束后能清理本局状态。

## 阶段 5：程序化关卡

目标：从固定房间切换到可重复游玩的楼层生成。

- [ ] 创建 RoomData。
- [ ] 创建 RoomPool。
- [ ] 创建 FloorGenerator。
- [ ] 用随机种子生成房间图。
- [ ] 支持 Combat、Elite、Treasure、Shop、Boss 等房间类型。
- [ ] 使用手工房间块拼接或 Level Streaming 加载房间。
- [ ] 保证入口、出口、出生点和刷怪点正确。

验收标准：

- 同一个种子生成相同楼层。
- 不同种子生成不同房间序列。
- 一层包含起点、若干战斗房和 Boss 房。

## 阶段 6：Meta 进度

目标：支持跨 run 的永久成长和内容解锁。

- [ ] 创建 SaveGame 数据结构。
- [ ] 保存已解锁角色、relic、技能池。
- [ ] 保存永久货币。
- [ ] 保存基础统计：死亡次数、通关次数、最远楼层。
- [ ] 让 Meta 解锁影响奖励池和角色选择。
- [ ] 添加保存/读取/重置调试命令。

验收标准：

- 退出游戏后再进入，永久进度保留。
- 解锁内容能进入后续 run 的候选池。
- 重置存档后恢复初始状态。

## 阶段 7：内容扩展与体验打磨

目标：从原型转向可玩 demo。

- [ ] 增加 5 到 10 个主动技能。
- [ ] 增加 20 个以上 relic。
- [ ] 增加至少 3 类敌人。
- [ ] 增加 Boss 原型。
- [ ] 增加基础 UI：血量、能量、技能栏、relic 列表。
- [ ] 增加基础音效和命中特效。
- [ ] 做一次完整 10 到 15 分钟 run。

验收标准：

- 玩家可以完整开始、推进、失败或通关一局。
- 技能和 relic 能形成至少几种明显构筑。
- 核心战斗手感达到可继续投入内容的程度。

## 延后评估项

以下内容不要在早期实现，等出现真实需求后再评估：

- GameFeature 插件化内容包。
- Experience 式模式配置。
- CommonUI。
- 多人联机和复制预测。
- 复杂存档云同步。
- 完全程序化几何生成。

## 当前优先级

最高优先级是阶段 1 的 GAS 技能闭环。只要技能释放、冷却、消耗、伤害、状态这些核心机制没跑通，就不要提前扩关卡生成和 meta 系统。
