## 页面命名规范

- ID 即文件路径（不含 `Docs/` 前缀和 `.md` 后缀）：
  - 例：文件 `Docs/20-modules/cpp/CombatCharacter.md` → id `20-modules/cpp/CombatCharacter`
- 命名风格：
  - `20-modules/cpp/` 下用**类名原样**（PascalCase，匹配 C++ 类）
  - `20-modules/python/` 下用**点分模块名**（如 `Tools.ue_tools_for_ai.ue_actions_executor`）
  - `30-tutorials/` 下用 **NN-kebab-slug**（2 位序号 + slug，如 `01-world-subsystem-lifecycle`）
  - 其他用 **kebab-case** 短 slug（如 `ai-agent-driven-dev`）
  - ADR 用 `NNNN-kebab-slug`（4 位数字 + slug）

## 页面格式（强制 frontmatter）

每个 wiki 页（不含 `_raw/`、`index.md`、`log.md`、`overview.md`、`README.md`、`00-meta/` 下文档）**必须**有 frontmatter：

### 通用页面示例

```yaml
---
id: 20-modules/cpp/CombatCharacter         # 等于文件路径（去前后缀）
type: module                                # module | subsystem | adr | runbook | topic | source | tutorial | guide | case-study
status: current                             # current | draft | stale | deprecated
language: zh                                # 当前固定 zh
owner: ai                                   # ai | human | mixed
anchors:                                    # 锚到代码（v1.0 只记 path）
  - path: Source/LyraStarterGame/Variant_Combat/CombatCharacter.h
  - path: Source/LyraStarterGame/Variant_Combat/CombatCharacter.cpp
related:                                    # 相关 wiki 页
  - "[[10-architecture/subsystems/combat]]"
sources:                                    # 来源 raw 素材
  - "[[_raw/specs/2026-05-13-combat-redesign]]"
last_synced: 2026-05-17                     # 最后一次 sync-anchors 时间
last_verified: 2026-05-17                   # 最后一次内容被实际 verify
tags: [combat, character, gameplay]
---
```

### 教程系列元数据（`_series.yaml`）

每个教程系列目录下必须有 `_series.yaml`：

```yaml
id: gas
title: "Gameplay Ability System 从入门到实战"
description: "系统学习 GAS 的核心概念、Ability 生命周期、Effect 与 Cue"
difficulty_range: [beginner, advanced]
estimated_hours: 12
lessons:
  - 01-gas-overview
  - 02-ability-activation
  - 03-gameplay-effects
  - 04-gameplay-cues
  # ...按实际课程扩展
tags: [gas, ability, gameplay]
```

### 字段说明

- `type`：决定模板与 lint 规则
  - `module`、`subsystem`：代码级文档，必须有 `anchors`
  - `adr`、`topic`：可无 `anchors`
  - `tutorial`、`guide`、`reference`、`case-study`：**不要求** `anchors`，鼓励填写 `engine_sources` / `lyra_sources`
  - `runbook`、`source`：操作手册/外部来源，按实际情况决定
- `status`：`stale` / `deprecated` 的页**不准被引用**而不警告
- `anchors`：`module` 和 `subsystem` 类型必填至少 1 条。v1.1 加 `sha256` 子字段
- `series` / `lesson_index`：`tutorial` 类型必填，标识所属系列和顺序
- `difficulty`：`tutorial` / `guide` 类型必填（beginner / intermediate / advanced）
- `prerequisites`：前置依赖的 wikilink 列表（tutorial/guide 建议填写）
- `engine_sources`：引用 UE 引擎源码路径 + 上下文说明（不触发 stale 检测）
- `lyra_sources`：引用 Lyra 项目源码路径 + 上下文说明
- `last_synced`：**所有页面必填**。创建时填写当天日期；内容或锚点变更时更新；v1.1 起也可由 `sync-anchors` 工作流自动更新
- `last_verified`：**所有页面必填**。创建时填写当天日期；人工/AI 实际复核内容准确性后手动更新，与 `last_synced` 不同

## Ingest 规则

详见 `[[../.codebuddy/skills/project-wiki/workflows/ingest]]` 工作流。要点：

1. 先查重（近似 70% 重合的页 → update，否则 create）
2. 按素材性质选层级（tutorials/architecture/modules/decisions/runbooks/references/topics/gotchas）
3. 必须挂 anchor（除非 type 是 `tutorial` / `guide` / `reference` / `case-study` / `topic` / `adr`）
4. 更新 `index.md` 与 `log.md`

## Query 规则

详见 `[[../.codebuddy/skills/project-wiki/workflows/query]]` 工作流。要点：

1. **任何项目任务前**先走 query 工作流
2. 三层漏斗：`index.md` → frontmatter grep → 全文 rg
3. 标注引用，发现 stale 必须警告

## Lint 规则

详见 `[[../.codebuddy/skills/project-wiki/workflows/lint]]` 工作流。v1.0 检查项：

1. wikilink 断链
2. 孤儿页
3. 缺 frontmatter 关键字段
4. status 一致性
5. index.md 一致性
6. 重复/近似页
7. tutorial 系列连续性（lesson_index 无间断）
8. `_series.yaml` 与实际课程文件一致性

v1.1 计划新增：anchor sha256 重算 → 自动标 stale。
