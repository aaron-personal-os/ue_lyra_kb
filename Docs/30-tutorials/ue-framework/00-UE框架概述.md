# UE框架概述

## 概述

> UE（Unreal Engine）框架是一套完整的游戏开发架构，采用**分层设计**思想，将游戏系统划分为**引擎层（EngineObjects）**和**游戏世界层（GameObjects）**两大层次。本文档作为 UE 框架技术专项文档的入口，提供整体架构概览和核心概念说明。

---

## 核心概念

### EngineObjects vs GameObjects

UE 框架将游戏系统分为两个层次，分别对应不同的生命周期和职责范围：

```mermaid
graph TB
    subgraph EngineObjects["引擎层（EngineObjects）"]
        Engine["UEngine<br/>引擎总控"]
        GameInstance["UGameInstance<br/>游戏实例"]
        GameViewportClient["UGameViewportClient<br/>游戏视口"]
        LocalPlayer["ULocalPlayer<br/>本地玩家"]
    end
    
    subgraph GameObjects["游戏世界层（GameObjects）"]
        World["UWorld<br/>游戏世界"]
        Level["ULevel<br/>关卡"]
        Actor["AActor<br/>游戏对象"]
        Component["UActorComponent<br/>组件"]
        GameMode["AGameModeBase<br/>游戏模式"]
        GameState["AGameStateBase<br/>游戏状态"]
        PlayerController["APlayerController<br/>玩家控制器"]
        Pawn["APawn<br/>角色"]
        PlayerState["APlayerState<br/>玩家状态"]
    end
    
    Engine --> GameInstance
    Engine --> GameViewportClient
    GameInstance --> LocalPlayer
    GameInstance --> World
    World --> Level
    World --> GameMode
    World --> GameState
    Level --> Actor
    Actor --> Component
    GameMode --> PlayerController
    PlayerController --> Pawn
    PlayerController --> PlayerState
    
    style EngineObjects fill:#e1f5fe
    style GameObjects fill:#f3e5f5
```

#### 引擎层（EngineObjects）

**生命周期**：引擎层面，在 World 创建之前就存在，独立于 World 之外。

| 类名 | 职责 | 创建时机 |
|------|------|----------|
| `UEngine` | 引擎总控，负责初始化 GameInstance、GameViewport、LocalPlayer，驱动游戏主循环 | 引擎启动 |
| `UGameInstance` | 游戏实例，管理跨 Level/World 的全局数据，管理 LocalPlayer 列表 | Engine::Init() |
| `UGameViewportClient` | 游戏视口，提供渲染、输入、UI 的高级接口 | Engine::Init() |
| `ULocalPlayer` | 本地玩家，代表坐在屏幕前的用户，持有 PlayerController | GameViewportClient 初始化 |

**特点**：
- ✅ 独立于 World，跨地图持久存在
- ✅ 负责引擎级别的初始化和管理
- ✅ 处理渲染、输入、本地玩家管理

#### 游戏世界层（GameObjects）

**生命周期**：World 层面，在 World 创建后才生成，属于 World 内部的对象。

| 类名 | 职责 | 创建时机 |
|------|------|----------|
| `UWorld` | 游戏世界，管理 Level、Actor、GameMode 等 | LoadMap() |
| `ULevel` | 关卡，包含多个 Actor | World 初始化 |
| `AActor` | 游戏对象，UE 中最基本的游戏实体 | 关卡加载/运行时生成 |
| `UActorComponent` | 组件，赋予 Actor 各种能力和特性 | Actor 创建时 |
| `AGameModeBase` | 游戏模式，定义游戏规则和玩法 | World::SetGameMode() |
| `AGameStateBase` | 游戏状态，保存游戏状态数据并复制同步 | GameMode::PreInitializeComponents() |
| `APlayerController` | 玩家控制器，控制 Pawn 的行为 | GameMode::Login() |
| `APawn` | 角色，玩家在游戏中的实体表示 | GameMode::PostLogin() |
| `APlayerState` | 玩家状态，存放玩家游戏过程中的数据 | PlayerController::PostInitializeComponents() |

**特点**：
- ✅ 依赖于 World，随 World 销毁
- ✅ 负责游戏逻辑的实现
- ✅ 支持网络复制（Replication）

## 执行流程

### 游戏主循环：Init → Tick → Exit

UE 游戏的执行本质是一个大循环，遵循 **初始化 → 主循环 → 退出** 的经典模式：

```mermaid
sequenceDiagram
    participant Launch as Launch Module
    participant GM as GuardedMain()
    participant EL as FEngineLoop
    participant Engine as UEngine
    participant World as UWorld
    participant Actors as AActor/Component
    
    Launch->>GM: 平台入口（WinMain/main）
    GM->>EL: PreInit()
    EL->>EL: LoadStartupModules()
    EL->>Engine: Init()
    Engine->>Engine: 创建 GameInstance
    Engine->>Engine: 创建 GameViewportClient
    Engine->>Engine: 创建 LocalPlayer
    Engine->>EL: Start()
    EL->>Engine: LoadMap()
    Engine->>World: 创建 UWorld
    Engine->>World: SetGameMode()
    World->>World: 创建 GameMode
    World->>World: 创建 GameState
    World->>World: 加载 Level 和 Actors
    World->>World: BeginPlay()
    
    Note over GM,Actos: 游戏主循环
    loop while(!IsEngineExitRequested())
        GM->>EL: Tick()
        EL->>Engine: Tick()
        Engine->>World: Tick()
        World->>World: Tick Levels
        World->>Actors: Tick Actors/Components
        World->>World: BroadcastTickDispatch()
        World->>World: BroadcastTickFlush()
    end
    
    GM->>EL: Exit()
    EL->>Engine: Exit()
    Engine->>World: Destroy()
    EL->>EL: UnloadModulesAtShutdown()
```

**关键阶段说明**：

| 阶段 | 关键函数 | 主要工作 |
|------|----------|----------|
| **PreInit** | `FEngineLoop::PreInit()` | 加载模块（Module）、注册 UObject 类、初始化低级系统 |
| **Init** | `FEngineLoop::Init()` | 创建 Engine 对象、初始化 GameInstance、创建视口和本地玩家 |
| **Start** | `UEngine::Start()` | 加载默认地图、创建 World、初始化游戏世界 |
| **Tick** | `FEngineLoop::Tick()` | 驱动游戏主循环、更新所有 World/Actor/Component |
| **Exit** | `FEngineLoop::Exit()` | 销毁 World、卸载模块、释放资源 |

## 分屏机制

UE 支持同一个游戏客户端的分成多个玩家独立操作的分屏机制，这是理解 `LocalPlayer`、`PlayerController` 可能存在多个的关键。

```mermaid
graph TB
    subgraph Client["游戏客户端（同一个进程）"]
        GVC["UGameViewportClient<br/>主视口"]
        
        subgraph Splitscreen["分屏视口"]
            VP1["视口1<br/>LocalPlayer1<br/>PlayerController1<br/>Camera1<br/>HUD1"]
            VP2["视口2<br/>LocalPlayer2<br/>PlayerController2<br/>Camera2<br/>HUD2"]
            VP3["视口3<br/>LocalPlayer3<br/>PlayerController3<br/>Camera3<br/>HUD3"]
            VP4["视口4<br/>LocalPlayer4<br/>PlayerController4<br/>Camera4<br/>HUD4"]
        end
        
        GVC --> VP1
        GVC --> VP2
        GVC --> VP3
        GVC --> VP4
    end
    
    subgraph World["同一个 UWorld"]
        Level["ULevel"]
        Actor1["Actor（玩家1）"]
        Actor2["Actor（玩家2）"]
        Actor3["Actor（玩家3）"]
        Actor4["Actor（玩家4）"]
    end
    
    VP1 --> Actor1
    VP2 --> Actor2
    VP3 --> Actor3
    VP4 --> Actor4
    
    style GVC fill:#ff9800
    style Splitscreen fill:#e1f5fe
    style World fill:#f3e5f5
```

**分屏机制特点**：
- ✅ 每个分屏视口代表一个独立的玩家
- ✅ 每个玩家有独立的 `LocalPlayer`、`PlayerController`、摄像机、HUD
- ✅ 所有玩家处于**同一个 World** 中
- ✅ 输入事件通过 `GameViewportClient` 路由到对应玩家的 `PlayerController`
- ✅ 分屏机制下也支持联网模式，共用一个物理连接（NetConnection），通过 `ChildConnection` 实现网络数据隔离

**典型应用**：
- 🎮 《双人成行》：双人分屏合作
- 🎮 《马里奥赛车》：四人同屏竞技

## 与其他模块的关系

UE 框架作为游戏开发的基础架构，与以下系统紧密相关：

```mermaid
graph TD
    Framework["UE 框架<br/>(本专项文档)"]
    GAS["Gameplay Ability System<br/>(GAS)"]
    StateTree["State Tree<br/>(行为树替代)"]
    Experience["Experience System<br/>(Lyra 核心机制)"]
    Network["网络同步系统<br/>(Replication/RPC)"]
    Input["Enhanced Input System<br/>(输入系统)"]
    UI["Common UI<br/>(UI 框架)"]
    
    Framework --> GAS
    Framework --> StateTree
    Framework --> Experience
    Framework --> Network
    Framework --> Input
    Framework --> UI
    
    style Framework fill:#ff9800,stroke:#333,stroke-width:4px
    style GAS fill:#e1f5fe
    style StateTree fill:#f3e5f5
    style Experience fill:#e8f5e9
    style Network fill:#fff3e0
    style Input fill:#fce4ec
    style UI fill:#f3e5f5
```

**关系说明**：

| 相关模块 | 关系 | 说明 |
|----------|------|------|
| **GAS** | GameMode 创建 AbilitySystem | Gameplay Ability System 依赖 GameMode/PlayerState 创建 AbilitySystemComponent |
| **State Tree** | AIController 使用 | UE5 新行为树系统，替代传统 Behavior Tree |
| **Experience** | GameMode 加载 Experience | Lyra 项目的核心机制，动态组合游戏玩法 |
| **网络同步** | GameState/PlayerState 复制 | 游戏状态通过网络复制同步到客户端 |
| **Enhanced Input** | PlayerController 设置输入 | UE5 新输入系统，替代传统 Input System |
| **Common UI** | GameViewportClient 显示 UI | UE 提供的跨平台 UI 框架 |
