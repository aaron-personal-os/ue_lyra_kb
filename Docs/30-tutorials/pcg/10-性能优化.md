# 性能优化

> **前置知识**：[09-高级技巧](./09-高级技巧.md)
> **预计阅读时间**：40 分钟

## 概念直觉

### PCG 性能的三大杀手

```
1. 点数量过多 → 内存爆炸
2. Draw Call 过多 → FPS 暴跌
3. 生成时间过长 → 卡顿
```

### 优化思路

**核心原则**：**少生成、晚生成、批量生成**。

```
少生成：降低密度、过滤无用点
晚生成：流式加载、按需生成
批量生成：实例化渲染、HISM
```

## 技术机制

### 1. LOD（细节层次）

**原理**：根据 **距离** 切换不同精度的模型。

```
距离 < 10m  → LOD0（高精度，10000 三角形）
距离 10-50m → LOD1（中精度，1000 三角形）
距离 > 50m → LOD2（低精度，100 三角形）
```

#### 配置 LOD

**步骤 1**：在 Static Mesh 编辑器中配置 LOD

1. 打开 Static Mesh（如 `Tree_01.FBX`）
2. `Details` → `LOD Settings` → `Number of LODs` = 3
3. 配置每个 LOD 的 **三角形数量**

**步骤 2**：在 PCG 中使用

```
[Static Mesh Spawner]
    Static Mesh = Tree_01 (已配置 LOD)
    Cull Distance = 5000
```

**关键发现**：
- HISM **自动切换 LOD**（根据距离）
- LOD 切换是 **平滑的**（不会突然 popping）
- 合理设置 LOD 可以减少 **50%+** 的三角形数量

---

### 2. 流式加载（Streaming）

**原理**：只生成 **玩家附近** 的内容，远处的内容 **不生成**。

#### 实现方式 1：多个 PCG Volume

```
Player 位置：
  - PCG Volume 1（中心，半径 100m）→ 生成
  - PCG Volume 2（半径 100-500m）→ 生成（低密度）
  - PCG Volume 3（半径 > 500m）→ 不生成
```

#### 实现方式 2：动态加载

```cpp
// 在 Player Controller 中
void AMyPlayerController::Tick(float DeltaSeconds)
{
    Super::Tick(DeltaSeconds);

    // 获取玩家位置
    FVector PlayerLocation = GetPawn()->GetActorLocation();

    // 查找附近的 PCG Volume
    TArray<AActor*> PCGVolumes = GetPCGVolumesInRadius(PlayerLocation, 1000.0f);

    // 生成附近的，卸载远的
    for (AActor* Volume : PCGVolumes)
    {
        float Distance = FVector::Dist(PlayerLocation, Volume->GetActorLocation());

        if (Distance < 500.0f)
        {
            // 生成
            UPCGComponent* PCGComponent = Volume->FindComponentByClass<UPCGComponent>();
            if (PCGComponent && !PCGComponent->IsGenerated())
            {
                PCGComponent->Generate();
            }
        }
        else if (Distance > 800.0f)
        {
            // 卸载
            UPCGComponent* PCGComponent = Volume->FindComponentByClass<UPCGComponent>();
            if (PCGComponent && PCGComponent->IsGenerated())
            {
                PCGComponent->Cleanup();
            }
        }
    }
}
```

### 3. 实例化渲染（HISM）

**原理**：多个实例 → 1 个 Draw Call。

#### HISM vs 独立 Actor

| 方式 | Draw Call | FPS（10000 棵树） |
|-----|-----------|-------------------|
| **独立 Actor** | 10000 | 10 FPS |
| **HISM** | 1 | 60 FPS |

#### 配置 HISM

```
[Static Mesh Spawner]
    bUseHISM = true (默认)
    Cull Distance = 5000
    bCastDynamicShadow = false (优化 shadow)
```

**关键发现**：
- 一个 HISM Component 只能渲染 **一种网格**
- 要渲染多种网格，需要 **多个 HISM Component**
- HISM 支持 **GPU Frustum Culling**（被剔除的实例不渲染）

### 4. 分块生成（Chunking）

**原理**：将大区域 **分块**，每块独立生成。

```
大区域（1km x 1km）：
  → 分成 100 块（100m x 100m）
  → 每块独立生成
  → 可以并行生成
```

#### 实现方式

**步骤 1**：创建 Sub Graph（生成单块）

```
[Surface Sampler] → [Transform Points] → [Static Mesh Spawner]
```

**步骤 2**：在主图表中调用 Sub Graph

```
[Grid Sampler] (100m x 100m 网格)
    ↓
[Loop] (遍历每个网格)
    ↓
[Call Sub Graph] (生成单块)
```

**优势**：
- **并行生成**（多块同时生成）
- **按需加载**（只生成玩家附近的块）
- **内存友好**（不用的块可以卸载）

### 5. 缓存机制

**原理**：避免 **重复计算**（相同输入 → 相同输出）。

#### PCG 内置缓存

**工作原理**：
1. 计算输入的 **哈希值**
2. 查找缓存（是否已生成）
3. 如果找到 → 直接返回缓存结果
4. 如果没找到 → 生成并缓存

**启用缓存**：

```
[PCG Component]
    bUseCache = true (默认)
    Cache Duration = 60.0 (缓存 60 秒)
```

#### 自定义缓存

**场景**：相同参数生成多次（如昼夜循环）。

```cpp
// 在自定义 Node 中
TArray<FPCGTaggedData> UPCGMyNodeSettings::Execute(
    const TArray<FPCGTaggedData>& InputData,
    const FPCGExecutionContext& Context) const
{
    // 1. 计算哈希
    uint32 Hash = ComputeHash(InputData, MyParameter);

    // 2. 查找缓存
    if (MyCache.Contains(Hash))
    {
        return MyCache[Hash]; // ← 直接返回
    }

    // 3. 没找到 → 生成
    TArray<FPCGTaggedData> OutputData = DoExpensiveCalculation(InputData);

    // 4. 缓存结果
    MyCache.Add(Hash, OutputData);

    return OutputData;
}
```

## 实践案例

### 案例 1：优化大规模森林（100 万棵树）

**问题**：生成 100 万棵树后，FPS 从 60 降到 10。

#### 优化步骤

**步骤 1**：使用 HISM

```
[Static Mesh Spawner]
    bUseHISM = true
    Cull Distance = 5000
```

**步骤 2**：配置 LOD

```
Tree_01:
  - LOD0 (距离 < 20m): 10000 三角形
  - LOD1 (距离 20-100m): 1000 三角形
  - LOD2 (距离 > 100m): 100 三角形
```

**步骤 3**：降低远处密度

```
[Surface Sampler]
    Density = 10.0 (近处，每立方米 10 棵)

[Surface Sampler 2]
    Density = 1.0 (远处，每立方米 1 棵)
```

**步骤 4**：分块生成

```
将 1km x 1km 区域分成 100 块（100m x 100m）
→ 每块独立生成
→ 只生成玩家附近的 9 块
```

**优化结果**：

| 指标 | 优化前 | 优化后 |
|-----|--------|--------|
| FPS | 10 | 60 |
| 内存 | 2 GB | 500 MB |
| 生成时间 | 60 秒 | 5 秒 |

### 案例 2：动态流式加载（开放世界）

**目标**：创建一个 10km x 10km 的开放世界，玩家可以自由探索。

#### 步骤 1：分块

```
将 10km x 10km 分成 10000 块（100m x 100m）
→ 每块一个 PCG Volume
```

#### 步骤 2：配置加载范围

```
加载范围：玩家周围 1km
卸载范围：玩家周围 > 2km
```

#### 步骤 3：实现动态加载

```cpp
// 在 Game Mode 中
void AMyGameMode::Tick(float DeltaSeconds)
{
    Super::Tick(DeltaSeconds);

    // 获取玩家位置
    FVector PlayerLocation = GetPlayerLocation();

    // 加载附近的块
    for (int32 X = -10; X <= 10; ++X)
    {
        for (int32 Y = -10; Y <= 10; ++Y)
        {
            FVector ChunkLocation = PlayerLocation + FVector(X * 100, Y * 100, 0);

            // 生成
            GeneratePCGAtLocation(ChunkLocation);
        }
    }

    // 卸载远的块
    for (int32 X = -20; X <= 20; ++X)
    {
        for (int32 Y = -20; Y <= 20; ++Y)
        {
            if (FMath::Abs(X) <= 10 && FMath::Abs(Y) <= 10) continue; // 跳过近处

            FVector ChunkLocation = PlayerLocation + FVector(X * 100, Y * 100, 0);

            // 卸载
            CleanupPCGAtLocation(ChunkLocation);
        }
    }
}
```

**预期结果**：
- 玩家永远看不到 "空白"（流式加载）
- 内存占用稳定（只加载附近）
- FPS 稳定（60 FPS）

## 常见错误

### Error 1：HISM 没有生效

**症状**：启用了 `bUseHISM = true`，但 Draw Call 仍然很高。

**原因**：
1. `Static Mesh Spawner` 没有使用 HISM
2. 每个树都是 **独立 Actor**（没有实例化）

**解决**：
1. 检查 `bUseHISM = true`
2. 确保 `Static Mesh Spawner` 是 **终端节点**（不输出数据）
3. 使用 `Debug` → `GPU Visualizer` 查看 Draw Call

### Error 2：流式加载卡顿

**症状**：玩家移动时，游戏卡顿（加载新块）。

**原因**：
1. 加载新块时 **阻塞主线程**
2. 生成时间过长（> 100ms）

**解决**：
1. 使用 **异步生成**（`Async Generate`）
2. 降低 `Density`（减少点数）
3. 预加载（提前生成玩家附近的块）

### Error 3：缓存不生效

**症状**：相同参数重复生成，缓存没有命中。

**原因**：
1. 输入数据 **哈希值不同**（如包含随机 Seed）
2. 缓存被 **意外清除**

**解决**：
1. 确保 `Seed` 固定（不随机）
2. 检查 `Cache Duration`（不要设置太短）
3. 使用 `PCG Profiler` 查看缓存命中率

## 性能分析工具

### 1. PCG Profiler

**用途**：分析每个 Node 的执行时间。

**使用方法**：
1. `Window` → `Developer Tools` → `PCG Profiler`
2. 点击 `Generate`
3. 查看每个 Node 的耗时

**优化建议**：
- 耗时 > 10ms 的 Node 需要优化
- 使用 **GPU 加速**（如果支持）
- 分块生成（降低单块耗时）

### 2. GPU Visualizer

**用途**：查看 Draw Call 数量。

**使用方法**：
1. 运行游戏
2. `Window` → `Developer Tools` → `GPU Visualizer`
3. 查看 `Draw Calls` 数量

**优化建议**：
- Draw Calls > 1000 → 使用 HISM
- Draw Calls > 5000 → 使用 **静态合批**

### 3. Unreal Insights

**用途**：分析 **CPU 性能**（哪些函数耗时最多）。

**使用方法**：
1. 启用 `Unreal Insights`（`-trace=cpu,gpu`）
2. 运行游戏
3. 打开 `Unreal Insights` 查看分析结果

**优化建议**：
- 找到 **最耗时的函数**
- 优化或缓存这些函数
