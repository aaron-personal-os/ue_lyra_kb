# 高级技巧

> **前置知识**：[08-生物群系创建](./08-生物群系创建.md)
> **预计阅读时间**：45 分钟

## 概念直觉

### 从"静态生成"到"动态世界"

**基础 PCG**：
```
编辑时生成 → 烘焙为静态资产 → 无法改变
```

**高级 PCG**：
```
运行时生成 → 动态更新 → 交互式世界
```

### 高级技巧的应用场景

| 场景 | 技术 | 示例 |
|-----|------|------|
| **大规模生成** | GPU 加速 | 100 万棵树 |
| **自定义逻辑** | C++ Node | 植物竞争模拟 |
| **动态世界** | 运行时生成 | 玩家砍树后重新生长 |
| **交互式** | Blueprint 集成 | 玩家行走压弯草地 |

## 技术机制

### 1. GPU 加速

**原理**：使用 **GPU Compute Shader** 执行 PCG 计算（而不是 CPU）。

**优势**：
- **并行度高**：GPU 有数千个核心
- **适合大规模**：100 万+ 个点
- **降低 CPU 占用**：CPU 可以做其他事情

**限制**：
- **只支持特定 Node**（Surface Sampler、Transform Points）
- **调试困难**（无法单步调试 GPU 代码）
- **平台限制**（移动平台可能不支持）

#### 启用 GPU 加速

**步骤 1**：在 PCG Graph 中启用

1. 打开 PCG Graph
2. `Details` 面板 → `Performance` → 勾选 `Use GPU`

**步骤 2**：确保 Node 支持 GPU

```
支持 GPU 的 Node：
  ✅ Surface Sampler
  ✅ Transform Points
  ✅ Filter Points

不支持 GPU 的 Node：
  ❌ Static Mesh Spawner（需要 CPU）
  ❌ Debug Draw（调试用）
```

#### 源码分析

**GPU 执行流程**（简化版）：

```cpp
// PCGGraph.cpp
void UPCGGraph::Execute(FPCGExecutionContext& InContext)
{
    if (bUseGPU)
    {
        // 1. 创建 GPU 缓冲区
        FPCGGPUBuffer InputBuffer = CreateGPUBuffer(InputData);

        // 2. 调度 Compute Shader
        DispatchComputeShader(InputBuffer, OutputBuffer);

        // 3. 读取结果
        TArray<FPCGTaggedData> OutputData = ReadGPUBuffer(OutputBuffer);

        return OutputData;
    }
    else
    {
        // CPU 执行（默认）
        // ...
    }
}
```

**关键发现**：
- GPU 加速需要 **DX12** 或 **Vulkan**（现代图形 API）
- `Static Mesh Spawner` 必须在 **CPU** 执行（GPU 无法创建 Actor）
- 可以 **混合使用**（部分 Node GPU，部分 CPU）

### 2. 自定义 Node（C++）

**应用场景**：
- 内置 Node 无法满足需求
- 需要复杂的自定义逻辑
- 需要访问 **外部数据**（如天气系统）

#### 创建自定义 Node

**步骤 1**：创建 Settings 类

```cpp
// PCG_CustomNodeSettings.h
UCLASS()
class UPCGCustomNodeSettings : public UPCGSettings
{
    GENERATED_BODY()

public:
    // 自定义参数
    UPROPERTY(EditAnywhere, Category = "Custom")
    float MyParameter = 1.0f;

    UPROPERTY(EditAnywhere, Category = "Custom")
    TSoftObjectPtr<UTexture2D> MyTexture;

protected:
    virtual TArray<FPCGPinProperties> GetInputPinProperties() const override;
    virtual TArray<FPCGPinProperties> GetOutputPinProperties() const override;

    virtual TArray<FPCGTaggedData> Execute(
        const TArray<FPCGTaggedData>& InputData,
        const FPCGExecutionContext& Context) const override;
};
```

**步骤 2**：实现 Execute 方法

```cpp
// PCG_CustomNodeSettings.cpp
TArray<FPCGTaggedData> UPCGCustomNodeSettings::Execute(
    const TArray<FPCGTaggedData>& InputData,
    const FPCGExecutionContext& Context) const
{
    TArray<FPCGTaggedData> OutputData;

    for (const FPCGTaggedData& TaggedData : InputData)
    {
        const UPCGPointData* PointData = Cast<UPCGPointData>(TaggedData.Data);
        if (!PointData) continue;

        // 创建新的点数据
        UPCGPointData* NewPointData = NewObject<UPCGPointData>();

        // 自定义逻辑：根据 MyParameter 过滤点
        for (const FPCGPoint& Point : PointData->GetPoints())
        {
            if (Point.Density > MyParameter)
            {
                NewPointData->AddPoint(Point);
            }
        }

        // 添加到输出
        FPCGTaggedData NewTaggedData = TaggedData;
        NewTaggedData.Data = NewPointData;
        OutputData.Add(NewTaggedData);
    }

    return OutputData;
}
```

**步骤 3**：注册 Node

```cpp
// 在模块启动时注册
PCG_REGISTER_SETTINGS(UPCGCustomNodeSettings, "Custom/My Custom Node")
```

**步骤 4**：在 PCG 图表中使用

1. 打开 PCG Graph
2. 右键 → 搜索 `My Custom Node`
3. 连接并使用

### 3. 运行时生成

**应用场景**：
- 开放世界（流式加载）
- 动态地形（玩家改造地形）
- 季节性变化（冬天落叶）

#### 运行时调用 Generate()

**方法 1**：通过 Blueprint

```
Event BeginPlay → Get PCG Component → Call Generate
```

**方法 2**：通过 C++

```cpp
// MyActor.cpp
void AMyActor::BeginPlay()
{
    Super::BeginPlay();

    // 获取 PCG Component
    UPCGComponent* PCGComponent = FindComponentByClass<UPCGComponent>();
    if (PCGComponent)
    {
        // 调用 Generate
        PCGComponent->Generate();
    }
}
```

#### 动态更新参数

**目标**：根据游戏状态动态调整 PCG 参数。

```cpp
// 获取 PCG Graph 的 Settings
UPCGGraph* Graph = PCGComponent->GetGraph();
if (Graph)
{
    // 查找 Node
    for (UPCGNode* Node : Graph->GetNodes())
    {
        if (UPCGSurfaceSamplerSettings* Settings = Cast<UPCGSurfaceSamplerSettings>(Node->GetSettings()))
        {
            // 动态修改 Density
            Settings->Density = 2.0f;

            // 重新生成
            PCGComponent->Generate(true); // bInForce = true
        }
    }
}
```

### 4. 与其他系统交互

#### 与 GAS 交互

**目标**：根据 Gameplay Effect 改变 PCG 生成。

**示例**：玩家使用"肥料技能" → 植物生长加速。

```cpp
// 在 Gameplay Effect 的 Execute 中
void UMyGameplayEffect::Execute()
{
    // 获取附近的 PCG Component
    UPCGComponent* PCGComponent = GetPCGComponentInRadius(500.0f);
    if (PCGComponent)
    {
        // 修改参数
        UPCGGraph* Graph = PCGComponent->GetGraph();
        // ... 修改 Density、Scale 等

        // 重新生成
        PCGComponent->Generate(true);
    }
}
```

#### 与 StateTree 交互

**目标**：根据 AI 状态改变 PCG 生成。

**示例**：AI 进入"巡逻"状态 → 生成警告标志。

```
StateTree Task:
    1. 获取 AI 位置
    2. 在位置周围生成 PCG（警告标志）
    3. 等待 5 秒
    4. 清除 PCG
```

## 实践案例

### 案例 1：GPU 加速的大规模森林

**目标**：生成 100 万棵树（使用 GPU 加速）。

#### 步骤 1：启用 GPU 加速

1. 打开 PCG Graph
2. `Details` → `Performance` → 勾选 `Use GPU`

#### 步骤 2：配置 Surface Sampler

```
[Surface Sampler]
    Density = 10.0 (每立方米 10 棵树)
    bUseGPU = true
```

#### 步骤 3：配置 Transform Points

```
[Transform Points]
    bUseGPU = true
    Scale Min = (0.8, 0.8, 0.8)
    Scale Max = (1.2, 1.2, 1.2)
```

#### 步骤 4：CPU 生成（Static Mesh Spawner）

```
[Static Mesh Spawner]
    bUseGPU = false (必须 CPU)
    Static Mesh = Tree_01
```

#### 步骤 5：测试

**预期结果**：
- Generate 时间 < 5 秒（GPU 加速）
- 100 万棵树生成
- FPS > 30（HISM 优化）

**目标**：玩家砍树后，树消失，5 秒后重新生长。

## 常见错误

### Error 1：GPU 加速不工作

**症状**：启用了 `Use GPU`，但性能没有提升。

**原因**：
1. Node 不支持 GPU
2. 图形 API 不支持（如 DX11）
3. 平台不支持（如移动平台）

**解决**：
1. 检查 Node 是否支持 GPU（查看文档）
2. 切换到 DX12 或 Vulkan
3. 在 PC 上测试（移动平台不支持）

### Error 2：运行时生成卡顿

**症状**：调用 `Generate()` 时，游戏卡顿 1-2 秒。

**原因**：
1. 生成点数过多（>10000）
2. 在主线程执行（阻塞游戏）

**解决**：
1. 降低 `Density`
2. 使用 **异步生成**（后台线程）
3. 分块生成（多个 PCG Volume）

### Error 3：自定义 Node 不显示

**症状**：创建了自定义 Node，但在 PCG 图表中找不到。

**原因**：
1. 没有注册 Node
2. 模块依赖没有配置

**解决**：
1. 检查 `PCG_REGISTER_SETTINGS` 宏
2. 检查 `.Build.cs` 文件（添加 PCG 依赖）
3. 重新编译项目

## 总结

通过本篇你学到了：

1. **GPU 加速** — 在 PCGGraph 中启用 `bUseGPU = true`，适合大规模采样
2. **自定义 Node** — 继承 `UPCGSettings`，实现 `Execute()` 函数
3. **运行时生成** — 使用 `GenerateOnDemand` 或 C++ 调用 `Generate()`
4. **异步生成** — 在后台线程执行，避免阻塞主线程
