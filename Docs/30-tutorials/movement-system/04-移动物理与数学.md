# 移动物理与数学

> 深入理解 CMC 移动参数的数学原理：摩擦力、加速度、减速度、空中控制。

## 概述

`CalcVelocity()` 是 CMC 的物理核心。它根据 `Acceleration`、`Friction`、`BrakingDeceleration` 计算出新 `Velocity`。本课用数学公式拆解每个参数的作用。

学完本课你将能够：
- 写出"有输入时"和"无输入时"的速度计算公式
- 解释 `Friction` 与 `BrakingFriction` 的区别
- 理解 `AirControl` 在空中控制中的具体效果
- 用数字说明"为什么按下 W 后角色不是瞬间加速到最高速"

---

## 一、CalcVelocity() 数学拆解

### 1.1 有输入时（Acceleration ≠ 0）

```cpp
// Engine/Source/Runtime/Engine/Private/Components/CharacterMovementComponent.cpp:L3860-L3866
if (!bZeroAcceleration)
{
    const float NewMaxInputSpeed = IsExceedingMaxSpeed(MaxInputSpeed) ? Velocity.Size() : MaxInputSpeed;
    Velocity += Acceleration * DeltaTime;
    Velocity = Velocity.GetClampedToMaxSize(NewMaxInputSpeed);
}
```

**数学公式**：

```
V_new = Clamp(V_old + a * Δt, MaxSpeed)
```

其中：
- `a` = `Acceleration`（方向 + 大小）
- `MaxSpeed` = `MaxWalkSpeed * AnalogInputModifier`（受模拟摇杆按压程度影响）

**为什么不是瞬间加速到最高速？**

因为每帧只加 `a * Δt`。假设：
- `MaxAcceleration = 2048 cm/s²`
- `MaxWalkSpeed = 600 cm/s`
- `Δt = 0.0167s`（60 FPS）

则每帧速度增量：`2048 * 0.0167 ≈ 34 cm/s`。从 0 加速到 600 cm/s 需要约 **17-18 帧**（≈ 0.28 秒）。

### 1.2 无输入时（Acceleration = 0）— 摩擦力刹车

```cpp
// L3828-L3845
if ((bZeroAcceleration && bZeroRequestedAcceleration) || bVelocityOverMax)
{
    const float ActualBrakingFriction = (bUseSeparateBrakingFriction ? BrakingFriction : Friction);
    ApplyVelocityBraking(DeltaTime, ActualBrakingFriction, BrakingDeceleration);
}
```

`ApplyVelocityBraking()` 的核心逻辑：

```cpp
// 伪代码，基于 ApplyVelocityBraking()
void ApplyVelocityBraking(float DeltaTime, float Friction, float BrakingDeceleration)
{
    // [1] 速度相关阻力（摩擦力）
    // 这是"与速度成正比"的阻力
    FVector FrictionForce = -Velocity * Friction * DeltaTime;
    
    // [2] 常数量减速度（独立于速度大小）
    // 这是"恒定的减速力"，确保低速时也能停下来
    FVector ConstantBraking = -Velocity.GetSafeNormal() * BrakingDeceleration * DeltaTime;
    
    // [3] 合成
    Velocity += FrictionForce + ConstantBraking;
    
    // [4] 防止过冲（速度方向翻转）
    if (FVector::DotProduct(Velocity, OldVelocity) < 0)
    {
        Velocity = FVector::ZeroVector;  // 完全停下来
    }
}
```

**数学公式（仅考虑摩擦力时）**：

```
V_new = V_old * (1 - Friction * Δt)
```

**为什么 `BrakingFrictionFactor` 默认是 2.0？**

```cpp
// Engine/Source/Runtime/Engine/Classes/GameFramework/CharacterMovementComponent.h:L306
/**
 * Factor used to multiply actual value of friction used when braking.
 * Note this is 2 by default for historical reasons, a value of 1 gives the true drag equation.
 */
float BrakingFrictionFactor = 2.0f;
```

默认值为 2.0 是历史遗留——早期 UE 代码中摩擦力倍率为 2，后来为了兼容性保留了这个值。设为 1.0 才是"真实"的物理摩擦力公式。

## 二、关键参数详解

### 2.1 GroundFriction（地面摩擦力系数）

**作用**：影响"松开按键后多久停下来"。值越大，停得越快。

**取值范围**：`[0, ∞)`，但通常 `< 10`。

**典型值**：
- 冰面：`0.1`（非常滑，滑行很远）
- 普通地面：`8.0`（默认）
- 油面：`0.05`（极滑）

**与 `BrakingFriction` 的关系**：

| 参数 | 触发条件 | 公式中的角色 |
|--------|-----------|--------------|
| `GroundFriction` | `bUseSeparateBrakingFriction = false`（默认） | `实际摩擦力 = GroundFriction * BrakingFrictionFactor` |
| `BrakingFriction` | `bUseSeparateBrakingFriction = true` | `实际摩擦力 = BrakingFriction * BrakingFrictionFactor` |

### 2.2 MaxAcceleration（最大加速度）

**作用**：决定"按下按键后多快达到最高速"。值越大，加速越快。

**取值范围**：`[0, ∞)`。设为 0 则永远无法加速（只能靠初始速度）。

**典型值**：
- 精确平台跳跃：`800 cm/s²`（慢加速，需要提前按键）
- 爽快动作游戏：`2048 cm/s²`（快速响应）
- 载具：`5000+ cm/s²`（很快达到高速）

### 2.3 BrakingDecelerationWalking（走路刹车减速度）

**作用**：松开按键后，与速度无关的"恒定减速度"。确保即使 `Friction = 0`（冰面），角色也能停下来。

**取值范围**：`[0, ∞)`。设为 0 则在冰面上永远滑行（除非撞墙）。

**典型值**：
- 普通角色：`2048 cm/s²`
- 重载角色：`4000 cm/s²`（更重，更难保持动量）
- 冰面专用：`512 cm/s²`（滑行很远）

### 2.4 AirControl（空中控制系数）

```cpp
// Engine/Source/Runtime/Engine/Classes/GameFramework/CharacterMovementComponent.h:L358
/**
 * When falling, amount of lateral movement control available to the character.
 * 0 = no control, 1 = full control at max speed of MaxWalkSpeed.
 */
float AirControl = 0.3f;  // 默认值
```

**作用**：跳跃后，在空中时是否还能用输入改变水平速度。

**取值范围**：`[0.0, 1.0]`。

**效果**：
- `AirControl = 0.0`：跳跃后水平速度锁定为起跳时的速度（"扔石子"模型）
- `AirControl = 1.0`：空中可以完全控制水平速度（"飞行"模型）
- `AirControl = 0.3`（默认）：空中可以达到地面最高速的 30%

**AirControlBoostMultiplier**（空中控制加速倍率）：

```cpp
// 当水平速度 < AirControlBoostVelocityThreshold 时，
// AirControl 会被乘以 AirControlBoostMultiplier（默认 2.0）
// 帮助角色"启动"空中移动
if (Velocity.Size() < AirControlBoostVelocityThreshold)
{
    AirControlUsed = AirControl * AirControlBoostMultiplier;
}
```

这解释了为什么"在跳跃顶点按方向键，角色会先缓慢响应，然后突然获得控制力"——这是 `AirControlBoost` 的效果。

## 三、Lyra 中的参数配置

Lyra 通过 **Gameplay Ability** 动态修改移动参数。例如"减速"效果：

```cpp
// 示例：Lyra 中一个减速 GE 的配置
// GameplayEffect 减少 MaxWalkSpeed 到 50%
// 同时减少 AirControl 到 0.1（更难空中控制）

UGameplayEffect::Modifier:
    Attribute: ULyraCharacterMovementComponent::MaxWalkSpeedAttribute()
    ModifierMagnitude: -50%  // 减半
    
    Attribute: ULyraCharacterMovementComponent::AirControlAttribute()
    ModifierMagnitude: -70%  // 大幅降低空中控制
```

**Lyra 的特殊做法**：用 GAS 属性系统动态修改 CMC 参数，而不是在 C++ 中硬编码"如果角色被减速，则 XXX"。

## 四、参数调优建议

### 4.1 "我想要更爽快的移动感"

- ✅ 增加 `MaxAcceleration`（更快加速）
- ✅ 减少 `BrakingDecelerationWalking`（松开按键后滑行更远）
- ✅ 增加 `AirControl`（空中更可控）
- ✅ 增加 `JumpZVelocity`（跳得更高）

### 4.2 "我想要更精确的平台跳跃"

- ✅ 减少 `MaxAcceleration`（慢加速，需要提前按键）
- ✅ 增加 `BrakingDecelerationWalking`（松开按键立刻停下）
- ✅ 减少 `AirControl`（跳跃后不能空中变向，增加难度）
- ✅ 设置 `AirControl = 0.0`（完全真实物理）

### 4.3 "我想做冰面效果"

- ✅ `GroundFriction = 0.1`（非常滑）
- ✅ `BrakingDecelerationWalking = 512`（低速时也能停）
- ✅ `MaxWalkSpeedCrouched = 0`（蹲伏时在冰面上不能移动）

## 总结

| 参数 | 作用 | 默认值 | 典型调整方向 |
|------|------|---------|--------------|
| `GroundFriction` | 地面摩擦力（速度相关阻力） | `8.0` | 冰面↓，胶地↑ |
| `MaxAcceleration` | 最大加速度 | `2048` | 爽快↑，精确↓ |
| `BrakingDecelerationWalking` | 走路刹车减速度（常量） | `2048` | 冰面↓，重载↑ |
| `AirControl` | 空中控制系数 | `0.3` | 飞行↑，真实↓ |
| `JumpZVelocity` | 跳跃初速度 | `420` | 跳得更高↑ |
| `GravityScale` | 重力倍率 | `1.0` | 月球↓，地球=1 |
